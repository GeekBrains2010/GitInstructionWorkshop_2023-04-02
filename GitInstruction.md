# Инструкция по работе с git

## Что это и для чего нужна система контроля версий?

Git — распределённая система управления версиями. Проект был создан Линусом Торвальдсом для управления разработкой ядра Linux, первая версия выпущена 7 апреля 2005 года. На сегодняшний день его поддерживает Джунио Хамано.

Git применяется для управления версиями в рамках колоссального количества проектов по разработке ПО, как коммерческих, так и с открытым исходным кодом. Система используется множеством профессиональных разработчиков программного обеспечения.

1. Создать новую папку 
2. В VSCode открыть эту папку

4. Ввести команду <git add название файла> название файла можно выбрать через клавишу tab. если майлов много и надо добавить все <git add .>

### Инструкция по созданию pull request
Для работы с репозиторием к которому нет доступа на правку необходимо на сайте GitHub.com выполнить команду *fork*.

>![Кнопка fork](git_fork.png)

Далее работаем по инструкции из п. 9 как со своим репозиторием. Праивлом хорошего тона является создание отдельной ветки для правки файлов репозитория.

Отправить файлы обратно на GitHub.com следует командой:

>**git push --set --upstream origin имя_ветки**

После выполнения такой команды обновить страницу на GitHub.com. Должна появиться кнопка *Compare & pull request*.

>![Кнопка Compare & pull request](git_C&P_request.png)

После нажатия на кнопку внести комментарии и отправить владельцу репозитория созданные изменения.
